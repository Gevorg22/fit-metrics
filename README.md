# fitMetrics

**[Открыть приложение](https://fit-metrics-production.up.railway.app)**

Дневник тренировок и питания. Логинишься по одноразовому коду на почту, выбираешь упражнение из базы 1323 упражнений с GIF-анимациями, записываешь подходы с весом и повторениями. История, прогресс, личные рекорды, дневник питания с AI, рейтинг пользователей - всё в одном месте.

## Возможности

### Авторизация и доступ
* **OTP по почте** - вводишь почту, получаешь 6-значный код, вводишь его. Без паролей. Email регистронезависим.
* **Сессия 90 дней** - не нужно входить заново каждый раз.
* **Гостевой режим** - можно попробовать приложение без регистрации, тренировки не сохраняются.
* **PWA** - устанавливается на домашний экран телефона как нативное приложение. Баннер установки появляется автоматически.
* **Push-уведомления** - если не тренировался 5 дней, приложение напомнит об этом. Подписка управляется в профиле, статус сохраняется в БД.

### Тренировки
* **1323 упражнения с GIF-анимациями** из ExerciseGymGifsDB, все названия переведены на русский.
* **Поиск** по названию и группе мышц (Грудь, Спина, Ноги, Пресс, Плечи, Бицепс, Трицепс, Кардио).
* **Избранные упражнения** - жми на сердечко рядом с упражнением, чтобы добавить в избранное. Для авторизованных пользователей избранное хранится в БД, для гостей - в localStorage.
* **Кардио-режим** - для упражнений типа «Бег» форма переключается в режим «Время / Дистанция» вместо «Вес / Повторения».
* **Прогрессивная перегрузка** - перед подходами показывает результаты прошлой тренировки с этим упражнением: «В прошлый раз: 80кг × 8».
* **AI-советник** - персональный совет по весу и повторениям на следующую тренировку на основе истории последних 5 сессий (Groq Llama 3.3 70b).
* **Таймер тренировки** - реальное время с момента старта.
* **Шаблоны тренировок** - сохрани набор упражнений и повтори одним нажатием.
* **Повторить тренировку** - в истории можно нажать «Повторить тренировку» и все упражнения автоматически добавятся в новую сессию.
* **Удаление подхода** - случайно добавил лишний подход? Жми крестик рядом с ним.
* **Клавиатурная навигация** - Enter/Tab между полями веса и повторений для быстрого ввода.
* **Заметки** к тренировке.
* **Защита от пустой тренировки** - нельзя сохранить без упражнений.
* **Floating-кнопка «Завершить»** - на мобильных фиксированная кнопка завершения всегда под рукой.

### История и аналитика
* **История тренировок** - список с датой, длительностью, подходами. Раскрывается по нажатию. Фильтр по упражнению.
* **График прогресса** - нажми на упражнение в Личных рекордах **или прямо в истории тренировок** и увидишь как менялись вес и повторения.
* **Личные рекорды** - лучший вес по каждому упражнению + расчётный максимум на 1 повторение по формуле Эпли.
* **Стрик** - счётчик дней подряд с тренировками, рекорд за всё время.
* **Достижения** - 15 значков за milestone-ы: первая тренировка, серии дней, тоннаж, время начала и другие.
* **Карта мышц** - SVG силуэт тела. Два режима: тепловая карта нагрузки за 7 дней и карта восстановления.
* **Объём по группам мышц** - горизонтальный бар-чарт: топ-8 мышечных групп по подходам за 7 дней.
* **Календарь активности** - GitHub-style heatmap: видно регулярность тренировок за год.
* **Поделиться тренировкой** - генерирует карточку итогов.
* **Экспорт в CSV** - забрать всю историю тренировок.

### Питание

* **Дневник питания** - записывай приёмы пищи за любой день с навигацией по датам.
* **Норма калорий** - рассчитывается автоматически по формуле Миффлина-Сан Жеора с учётом умеренной активности (TDEE). Режим цели (похудение / набор / поддержание) определяется по сравнению текущего и целевого веса.
* **Рекомендуемые макросы** - суточная норма белков, жиров и углеводов под каждую цель.
* **AI-сканер еды** - сфотографируй блюдо, Groq Llama 4 Scout распознает его и посчитает калории и КБЖУ. Работает с несколькими блюдами на фото.
* **Ввод вручную с AI-оценкой** - введи название продукта и граммы, AI заполнит КБЖУ автоматически; или заполни поля самостоятельно.
* **Итог за день** - суммарные калории и макросы по всем записям с прогресс-баром к целевой норме.

### Рейтинг

* **Топ-10 пользователей** по трём категориям: суммарный объём (кг), количество тренировок, максимальный вес снаряда.
* **Медали** для первых трёх мест, текущий пользователь выделен в списке.
* **Статистика участника** - нажми на любого пользователя и увидишь его профиль: дата регистрации, количество тренировок, суммарный объём, время в зале, любимое упражнение.

### Вес и тело
* **Дневник веса** - ввод веса тела за день.
* **График** динамики веса с фильтром по периоду. **Линия целевого веса** отображается пунктиром если цель задана в профиле.
* **ИМТ** - рассчитывается автоматически по весу и росту, отображается на дашборде с цветовой категорией.
* **Прогресс к цели** - прогресс-бар «сколько осталось до целевого веса».

### Профиль
* **Имя пользователя** - редактируется inline, сохраняется в БД.
* **Пол** - переключатель мужской / женский, сохраняется в БД.
* **Параметры тела** - рост, целевой вес, дата рождения (рассчитывается возраст). Всё редактируется inline.
* **Статистика** - суммарный поднятый вес, среднее время тренировки, топ упражнений, количество тренировок.
* **Уведомления** - удобный Switch-тогл, статус сохраняется в БД.

### Интерфейс
* **Светлая и тёмная тема** - переключение в шапке, сохраняется между сессиями, нет вспышки при загрузке.
* Адаптивный дизайн для мобильных и десктоп.

## Локальный запуск

### 1. Клонируй репозиторий

```bash
git clone https://github.com/Gevorg22/fit-metrics.git
cd fit-metrics
```

### 2. Установи зависимости

```bash
npm install
```

### 3. Настрой переменные окружения

Создай `.env.local` и заполни:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/fitmetrics"
DIRECT_URL="postgresql://user:password@localhost:5432/fitmetrics"
AUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://localhost:3000"
RESEND_API_KEY="re_xxxxxxxxxx"
EMAIL_FROM="fitMetrics <noreply@yourdomain.com>"
```

Для AI-функций (советник по упражнениям, сканер еды):

```env
GROQ_API_KEY="gsk_xxxxxxxxxx"
```

Для push-уведомлений (опционально):

```env
NEXT_PUBLIC_VAPID_PUBLIC_KEY="your-vapid-public-key"
VAPID_PRIVATE_KEY="your-vapid-private-key"
```

### 4. Примени схему БД и засей упражнения

```bash
npm run db:push
npx tsx prisma/seed-gifdb.ts
```

### 5. Запусти сервер разработки

```bash
npm run dev
```

Открой [http://localhost:3000](http://localhost:3000).

## Структура проекта

```
fitMetrics/
├── app/
│   ├── (auth)/          # Страницы авторизации (login, verify-otp)
│   ├── (protected)/     # Защищённые страницы (dashboard, workout, history, profile)
│   ├── api/             # API routes
│   ├── providers.tsx    # SessionProvider, QueryClientProvider, Ant Design ConfigProvider
│   └── layout.tsx       # Корневой layout
├── components/
│   ├── dashboard/       # Главная: история, вес, рекорды, карта мышц, хитмап, BMI
│   ├── history/         # Страница истории тренировок
│   ├── layout/          # Навигация, футер
│   ├── leaderboard/     # Рейтинг пользователей, статистика участника
│   ├── nutrition/       # Дневник питания, сканер еды, норма калорий
│   ├── profile/         # Профиль: параметры тела, уведомления, пол
│   └── workout/         # Поиск упражнений, форма подходов, шаблоны
├── hooks/               # React Query хуки (streak, achievements, muscle, leaderboard и др.)
├── lib/
│   ├── api/             # Типизированные fetch-функции для всех endpoint-ов
│   ├── utils.ts         # Общие утилиты (форматирование, изображения, расчёты)
│   ├── api-response.ts  # Хелперы для унифицированных API-ответов
│   └── prisma.ts        # Prisma client
├── prisma/              # Схема БД, скрипты сидинга
├── stores/              # Zustand store (активная тренировка)
├── types/               # TypeScript типы
└── .github/workflows/   # GitHub Actions (бэкап БД)
```
